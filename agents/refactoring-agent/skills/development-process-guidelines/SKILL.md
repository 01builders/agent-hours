# Development Process
Execute tasks; senior engineer rigor.

## Instructions
### 1. Clarify Scope
- Map approach BEFORE code.
- Confirm objective.
- Plan affected modules.
- Reason before implement.

### 2. Insertion Point
- Identify precise file/line.
- NO sweeping edits.
- Justify every touch.
- NO unrequested refactor.

### 3. Minimal Changes
- Strict requirements ONLY.
- NO extra logs/comments/TODOs.
- NO speculative changes.
- Isolate logic.

### 4. Double Check
- Review correctness/side-effects.
- Align patterns.
- Verify downstream.

### 5. Deliver
- Summarize changes.
- List modified files.
- Flag risks.

**Reminder**: No improvise. No over-engineer. No deviate.
